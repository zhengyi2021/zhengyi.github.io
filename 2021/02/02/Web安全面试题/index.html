Web安全面试题

Web安全面试题

XSS

本质
攻击者向web页面中插入恶意JavaScript语句。

分类

  • 反射性xss
  • DOM型xss
  • 存储型xss

原理

  • 反射性xss原理
    1. 攻击者构造包含恶意JS语句的URL
    2. 受害者访问URL
    3. 网站从URL中获取数据作为响应页面的数据
    4. 受害者浏览器解析执行恶意代码
  • DOM型xss原理
    1. 攻击不经过服务器
    2. 利用网页的JS修改HTML
    3. 插入恶意脚本
  • 存储型xss原理
    1. 攻击者将恶意JS脚本长期保存在服务器端数据库中
    2. 用户一旦访问页面，就会执行恶意脚本

危害

  • 盗取用户cookie
  • 盗取账号
  • 进行未授权的操作
  • 修改DOM
  • 刷弹窗广告
  • xss蠕虫攻击
  • 劫持用户，渗透内网

防御手段

  • 一切用户输入皆不可信，输出时进行验证
  • 将HTML元素内容、属性、URL请求参数、CSS值进行编码
  • 若编码影响业务，则使用白名单规则进行检测和过滤
  • 使用W3C提出的CSP（内容安全策略），定义域名白名单
  • 设置Cookie的httpOnly属性

CSRF

本质
跨站请求伪造。攻击者诱使用户打开构造的网站，从而获取用户的登录状态向被攻击的网站发起跨站请求，即冒充该用户的身份凭证绕过后台的身份验证进行某些操作。

防御手段

  • 服务器端验证请求来源（Referer、Origin）
  • 使用CSRF Token（随机Token）
  • 加入二次验证（独立密码、验证码）

POC

Proof Of Concept，“观点证明”。这个短语会在漏洞报告中使用，漏洞报告中的POC则是一段说明或者一个攻击的样例，使得读者能够确认这个漏洞是真实存在的。仅起到验证作用

EXP

Exploit，”漏洞利用“。意思是一段对漏洞如何利用的详细说明或者一个演示的漏洞攻击代码，可以使得读者完全了解漏洞的机理以及利用的方法。

常见的CMS

  • ThinkPHP
  • Weblogic
  • dedeCMS
  • Struts2
  • WordPress

SQL注入

原理
通过构造特殊的SQL语句作为参数传入web应用程序，进而让服务器解析执行的攻击

危害

  • 未授权操作数据库数据
  • 篡改网页内容
  • 添加修改系统账号或数据库账号
  • 网站挂马

典型攻击手段

  1. 判断是否存在注入
  2. 收集信息，判断数据库类型
  3. 根据注入参数类型构造SQL语句
  4. 猜解数据库名，表名，字段名
  5. 收集数据库信息

MySQL的information_schema的结构
information_schema
—schemata
|—schema_name
—tables
|—table_schema,table_name
—columns
|—table_schema,table_name,column_name

文件上传

必要条件

  1. 有上传点
  2. 知道木马文件的访问路径

数据库服务开放且没有更改默认端口，但只能扫描到80端口的原因

  1. 站库分离
  2. 防火墙禁用3306端口
  3. 3306端口仅监听localhost

拿到webshell之后可以干什么

  1. 找数据库配置文件，搜集用户名密码
  2. 提权
  3. 内网探测
  4. 添加计划任务、修改注册表进行反弹shell
